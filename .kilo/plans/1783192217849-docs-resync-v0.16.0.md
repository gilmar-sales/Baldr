## Missing-feature documentation to add

1. **`docs/usage/static-files.md`** (new) — covers `app->MapStaticFiles(urlPrefix, fsPath)` (`src/Baldr/Application/WebApplication.hpp:60`). Note that `MapStaticFiles` routes are intentionally excluded from OpenAPI spec. Add nav entry under "Usage".

2. **`docs/usage/streaming-results.md`** (new) — covers `IStreamingResult`, `ChunkedStreamResult` (callback-driven), `FileStreamResult` (file-backed with `Content-Disposition: attachment`). Reference `examples/FileStream` for end-to-end usage. Note that framework auto-emits `Transfer-Encoding: chunked` and implementations MUST NOT set `Content-Length`/`Transfer-Encoding` themselves. Add nav entry under "Usage".

3. **`docs/extensions/compression.md`** (new) — `CompressionMiddleware` + `CompressionOptions` (`src/Baldr/Middleware/Compression/Middleware.hpp`). Document `mimeTypePrefixes`, `minBodyBytes`, `level` (zlib 1..9, -1 default). Note: requires zlib (system). Add nav entry under "Extensions".

4. **`docs/extensions/cors.md`** (new) — `CorsMiddleware` + `CorsOptions` (`src/Baldr/Middleware/Cors.hpp`). Document `allowOrigin`, `allowMethods`, `allowHeaders`, `allowCredentials`, `maxAge`. Note `OPTIONS` short-circuit. Add nav entry under "Extensions".

5. **`docs/extensions/csrf.md`** (new) — `CsrfMiddleware` + `CsrfOptions` (`src/Baldr/Middleware/Csrf.hpp`). Document double-submit cookie pattern (Angular convention `XSRF-TOKEN`/`X-XSRF-TOKEN`), `protectedMethods`, `exemptPathPrefixes`, `issueCookieOnSafeRequest`. Add nav entry under "Extensions".

6. **`docs/extensions/security-headers.md`** (new) — `SecurityHeadersMiddleware` + `SecurityHeadersOptions` (`src/Baldr/Middleware/SecurityHeaders.hpp`). Document each header toggle (X-Content-Type-Options, X-Frame-Options, Referrer-Policy, HSTS, Permissions-Policy, COOP/CORP/COEP). Add nav entry under "Extensions".

7. **`docs/extensions/exception-handler.md`** (new) — `ExceptionHandlerMiddleware` + `ExceptionHandlerOptions` (`src/Baldr/Middleware/ExceptionHandler.hpp`). Document `mapper`, `includeDetailsInDev`, `status`, `contentType`. Add nav entry under "Extensions".

8. **Per-example pages** — split `docs/authoring/examples.md` into one page per example under `docs/authoring/examples/`:

9. **`docs/usage/route-options.md`** — add a "Route groups" subsection (with `MapGroup` code excerpt) and a "Static files" cross-link to `static-files.md`.

## Open questions / risks

- **Reflection support**: docs/setup/build.md currently claims reflection is "internal, used by LoggingMiddleware" — this is no longer accurate after OpenAPI was added. If reviewers prefer, this can be re-worded to say reflection is used by the OpenAPI extension instead. Already addressed in the drift table.
- **zlib as a consumer requirement**: when Baldr is consumed via `FetchContent`, consumers will now also need zlib on the host system (CMakeLists forces `find_package(ZLIB)` and FATAL_ERRORs if missing). Worth flagging in the PR description so the maintainer can confirm consumers see a clear error.
- **Examples link rot**: per-example pages link to GitHub permalinks. If `examples/OpenApiExample/src/main.cpp` changes after the docs PR merges, those anchors will need updating. Acceptable — same risk exists today.
